# Product Direction

## 한 줄 정의

`clibase`는 `CLI-first automation workbench`다.

## 제품 북극성

사용자가 커스텀한 CLI를 중심으로 프로젝트를 통합 운영하고, GUI와 AI는 그 CLI와 같은 제어 계약 위에서 동작하는 작업환경을 만든다.

## 핵심 방향

- 글로벌 CLI가 전체 시스템의 기준 제어면이 된다.
- 각 프로젝트는 글로벌 CLI에서 파생된 자신의 프로젝트 CLI 컨텍스트를 가진다.
- GUI는 CLI를 대체하는 면이 아니라 설정, 가시화, 디버깅, 관찰을 돕는 면이다.
- AI는 별도 특권 기능이 아니라, 같은 CLI/action 시스템을 통해 모든 안정된 기능을 제어해야 한다.
- 새 기능이 추가되더라도 CLI projection과 action contract만 맞으면 AI와 자동화가 바로 따라붙을 수 있어야 한다.

## 누구를 위한 제품인가

- 주 사용자: 약간의 코드 지식이 있는 파워유저
- 비주 사용자: 개발자는 아니지만 CLI, 설정 파일, 프롬프트, 간단한 스크립트 개념은 이해하는 사람
- 제외 대상: 아무 코드 개념도 없는 완전한 일반 사용자

## 제품이 제공해야 하는 핵심 가치

1. 사용자는 글로벌 CLI를 기준으로 자신의 프로젝트 운영 방식을 표준화할 수 있어야 한다.
2. 사용자는 프로젝트마다 다른 CLI 파생 설정, 모듈, Skill, MCP, 채널, 자동화를 한곳에서 관리할 수 있어야 한다.
3. 사용자는 GUI를 통해 복잡한 설정과 상태를 더 쉽게 다룰 수 있어야 한다.
4. AI는 사용자가 만든 시스템을 새 기능이 생겨도 다시 배워서가 아니라 같은 제어 계약으로 계속 다룰 수 있어야 한다.
5. 비개발자에 가까운 사용자도 약간의 코드 지식만으로 자동화를 확장할 수 있어야 한다.

## 제품이 아닌 것

- GUI-first 앱 빌더가 아니다.
- 채팅창이 중심인 AI 앱이 아니다.
- 완전한 no-code 플랫폼이 아니다.
- 범용 IDE 복제품이 아니다.
- 각 기능이 제멋대로 명령 체계를 가지는 툴 모음이 아니다.

## 제품 중심축

### 1. CLI 중심

- 모든 안정된 기능은 CLI로 호출 가능해야 한다.
- GUI는 CLI를 우회하지 않고 같은 action catalog를 사용해야 한다.
- 프로젝트 CLI는 글로벌 CLI의 파생이다.

### 2. 프로젝트 중심

- 프로젝트는 저장 단위이자 운영 단위다.
- CLI, 모듈, 채널, auth, attachment, 로그가 프로젝트 안에서 함께 관리된다.

### 3. AI 제어 가능성

- AI가 시스템을 잘 쓰려면 새 기능도 같은 CLI/action 계층에 연결되어야 한다.
- AI 전용 숨은 API를 만들지 않는다.
- 새 기능은 `사람 CLI`와 `AI CLI`가 같은 계약을 공유해야 한다.

### 4. 반자동화에서 고급 자동화로의 확장

- 처음에는 사용자가 명령을 직접 실행한다.
- 이후 preset, project CLI, Skill, MCP, AI 조합으로 자동화를 점점 높인다.
- 제품은 이 확장 경로를 자연스럽게 제공해야 한다.

## 왜 GUI가 필요한가

- 프로젝트 구조와 설정을 한눈에 보고 편집하기 위해
- 브라우저, 터미널, 로그, 결과를 함께 관찰하기 위해
- 복잡한 CLI 파생 설정과 attachment를 시각적으로 관리하기 위해
- AI가 수행한 작업과 시스템 상태를 추적하기 위해

## 방향이 주는 구현 원칙

- 새 기능은 먼저 `무슨 action/CLI로 제어되는지`가 정의되어야 한다.
- 새 모듈은 `사람이 조작하는 UI` 이전에 `제어 가능한 contract`가 있어야 한다.
- GUI에만 있고 CLI/action에 없는 기능은 제품 방향에 맞지 않는다.
- AI가 제어할 수 없는 새 기능은 제품 완성으로 보지 않는다.

## 성공한 제품의 모습

- 사용자는 글로벌 CLI 한 세트를 기준으로 여러 프로젝트를 운영한다.
- 프로젝트마다 필요한 도구, 모듈, 채널, auth, Skill, MCP를 붙여도 전체 제어 방식은 흔들리지 않는다.
- GUI는 복잡함을 줄이고, CLI는 제어력을 유지하며, AI는 같은 시스템을 이해하고 자동화한다.
- 비개발자에 가까운 파워유저가 적은 코드 지식으로도 실전 자동화를 스스로 조립할 수 있다.
