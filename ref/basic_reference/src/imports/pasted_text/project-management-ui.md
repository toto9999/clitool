왼쪽 가장 바깥의 얇은 사이드바에는 맨 위에 프로젝트 관리 아이콘이 하나 고정되어 있다.
그 아래부터는 프로젝트 관리에서 생성해서 저장한 프로젝트들이 아이콘 형태로 순서대로 쌓인다.

여기서 중요한 건,
프로젝트 관리 탭은 실제 프로젝트 내부 작업 화면이 아니다.
이 화면은 프로젝트를 만드는 곳이자, 각 프로젝트에 들어갈 탭 구조, 탭별 모듈 구성, 배치 방식, UI 형태, 동작 방식을 설정하는 곳이다.

하나의 프로젝트에는 여러 개의 탭을 구성할 수 있어야 한다.
프로젝트는 상위 단위이고, 실제 작업 화면은 그 프로젝트 안에 속한 각 탭으로 나뉜다.
각 탭은 서로 독립적인 작업 단위이며, 탭마다 별도의 레이아웃과 모듈 구성을 가진다.

또한 모듈은 프로젝트 전용으로 새로 만드는 개념이 아니라, 공통으로 관리되는 툴을 프로젝트에 가져다 쓰는 구조여야 한다.
즉 브라우저, 터미널, PDF 뷰어 같은 모듈은 독립적으로 존재하고, 프로젝트 관리에서는 이 공통 모듈을 선택해서 각 탭에 배치한다.
같은 모듈을 하나의 프로젝트 안에 여러 개 넣을 수 있고, 같은 탭 안에서도 복수 인스턴스를 둘 수 있어야 한다.

즉 흐름은 이렇다.

핵심은 이게 내부 작업 화면 진입이 아니라 설정 편집이라는 점이다.

즉, 브라우저나 터미널이 실제로 실행되는 화면이 아니라
“이 프로젝트에는 Analysis 탭, Report 탭이 있고, Analysis 탭에는 브라우저 2개와 터미널 1개를 넣고, 좌우 분할로 배치하고, Report 탭에는 PDF 뷰어 하나만 전체 화면으로 둔다”
이런 구조를 결정하는 설계 화면이어야 한다.

프로젝트 저장 후 동작

프로젝트 관리에서 새 프로젝트를 만들거나 기존 프로젝트를 수정한 뒤 저장하면
해당 프로젝트가 확정되고, 왼쪽 전역 사이드바에 프로젝트 관리 아이콘 아래부터 프로젝트 아이콘이 생성된다.

저장 전에는 설계 중인 상태이고,
저장 후에는 실제 진입 가능한 프로젝트가 된다.

즉, 프로젝트 관리 탭은 프로젝트를 정의하는 영역이고,
왼쪽 사이드바에 생성된 프로젝트 아이콘은 그 정의 결과물을 실행하는 진입점이다.

프로젝트 진입 시 화면 구성

사용자가 왼쪽 전역 사이드바에서 특정 프로젝트 아이콘을 클릭하면 실제 프로젝트 작업 화면으로 진입한다.

이때 화면 구조는 다음과 같아야 한다.

가장 왼쪽 전역 사이드바

맨 위에는 프로젝트 관리 아이콘이 고정되어 있다.

그 아래에는 저장된 프로젝트 아이콘들이 세로로 정렬되어 있다.

이 영역은 프로젝트 전환용이다.

그 오른쪽의 프로젝트 전용 탭 사이드바

특정 프로젝트에 진입하면, 가장 왼쪽 사이드바의 바로 오른쪽에 해당 프로젝트의 탭 목록을 보여주는 사이드바가 하나 더 생겨야 한다.

이 사이드바에는 현재 프로젝트에 정의된 탭들이 표시된다.

예:
Analysis
Modeling
Report

사용자는 여기서 탭을 클릭해서 작업 화면을 전환한다.

이 영역은 모듈 목록이 아니라 탭 전환용 영역이다.

중앙 작업 영역

중앙 작업 영역은 현재 선택된 탭의 설정값에 따라 렌더링된다.

즉, 사용자가 Analysis 탭을 누르면 Analysis 탭에 정의된 레이아웃과 모듈 구성이 보이고,
Report 탭을 누르면 Report 탭에 정의된 다른 레이아웃과 모듈 구성이 보이게 된다.

탭 전환 동작

프로젝트에 진입한 뒤 사용자가 프로젝트 전용 탭 사이드바에서 탭 하나를 클릭하면
중앙 작업 영역이 해당 탭 기준으로 전환되어야 한다.

이때 적용되는 것은 해당 탭에 저장된 설정값이다.

즉 탭마다 아래 항목이 각각 독립적으로 적용된다.

레이아웃

모듈 개수

모듈 종류

모듈 위치

모듈 UI

모듈 동작

예를 들면
탭 A는 단일 브라우저 전체 화면
탭 B는 브라우저 1개 + 터미널 1개의 좌우 2분할
탭 C는 PDF 뷰어 + AI 모듈 + 터미널의 3분할
처럼 탭별로 완전히 다른 구성이 가능해야 한다.

탭별 UI 설정 범위

하나의 탭 안에 하나의 모듈만 둘 수도 있어야 하고,
두 개의 모듈, 세 개의 모듈, 네 개의 모듈을 둘 수도 있어야 한다.

즉, 탭 단위에서 아래와 같은 UI 설정이 가능해야 한다.

하나의 모듈만 전체 화면으로 사용하는 탭

두 개의 모듈을 좌우 또는 상하로 나누는 탭

세 개 이상의 모듈을 조합해서 사용하는 탭

동일 모듈을 복수 인스턴스로 배치하는 탭

결론적으로 탭은 단순 분류 기준이 아니라,
각각 독립적인 레이아웃과 모듈 구성을 가진 실제 작업 단위여야 한다.

한 줄로 정리하면

프로젝트 관리 탭은 프로젝트와 탭 구조를 설계하는 화면이고,
저장된 프로젝트는 왼쪽 사이드바에 생성되며,
프로젝트에 진입하면 그 오른쪽에 탭 전환용 사이드바가 나타나고,
각 탭마다 독립적인 레이아웃과 모듈 구성이 적용되는 구조다.
